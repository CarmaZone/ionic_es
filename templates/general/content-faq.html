Cuando trabajamos con Cordova e Ionic, Node se convertirá en tu mejor amigo. Especialmente trabajando con Node y la línea de comandos. A continuación se muestran algunos problemas que no son necesariamente de Ionic, pero son errores comunes relacionados con Cordova, Node, NPM y Android:

Android SDK

Instalar el SDK de Android es un proceso complicado en Windows y Linux.

Recientemente Ubuntu ha logrado una fantástica herramienta llamada Ubuntu Developer Tools Center con la cual podemos instalar el SDK de Android y Android Studio. No vamos a entrar en detalles, puedes seguir los pasos en el siguiente artículo del blog

El proceso de configuración para Android en Windows puede verse en profundidad enIonic Learn Site. Lo único que pienso que podría añadirse durante la configuración es la instalación de git.

En Macs suele haber menos problemas en la instalación del entorno de desarrollo para Android. Hay varias maneras de hacerlo, ésta es solo una de ellas. El objetivo finas es tener el SDK instalado, Java y ant.

Instlación de Java - puede dirigirte a ésta página. Instalación de Ant - La forma más rápida es usar Homebrew.

$ brew install ant

Instalar el SDK - Se puede hacer de varias maneras. La que más me gusta es instalar Android Studio.Esto de instalará un editor con el SDK de Android integrado. Ahora nuestro SDK está en nuesta carpeta de Aplicaciones

Edita .bash_profile - i ha seguido los pasos anterioes, necesitarás editar tu .bash_profile para incluir lo siguiente. Si no instalaste Android Studio no hay problema, necesitarás cambiar la ruta del SDK. Incluye el SDK tools y SDK Platform-tools.

vim ~/.bash_profile

Añade ésto:

export PATH=${PATH}:/Applications/Android\ Studio.app/sdk/platform-tools:/Applications/Android\ Studio.app/sdk/tools
export JAVA_HOME=$(/usr/libexec/java_home)
export PATH=${JAVA_HOME}/bin:$PATH
export PATH=/usr/local/bin:$PATH

Ahora, ejecuta source ~/.bash_profilepara actualizar los cambios.

Genymotion

Éste consejo no se puede enfatizar lo suficiente! Aunque trabajemos de forma nativa, el emulador de Android es un sufrimiento. Inicia lento, y se bloquea y deja de responder fácilmnente. Afortunadamente Genymotion es un estupendo reemplazo. Usa una máquina virtual para emular dispositivos android y lo hace perfectamente. A recordar, si quieres desplegar en un dispositivo en genymotion, tu ordenador lo tratará como un dispositivo real, por lo que en lugar de ejecutar

$ ionic emulate android

usa runen su lugar.

$ ionic run android

Versiones de node sin liberar

Si eres un desarrollador de node, probablemnete uses la última versión en tu sistema. Aunque esto es fantástico para testear aplicaciones con node, Cordova e Ionic necesitan estabilidad. Cuando hagas algo con la línea de comandos, usa siempre la versión estable de node

Empezando con el Simulador de iOS

Arrancar el emulador de IOS desde la línea de comandos es una característica realmente poderosa. Te libera de abrir Xcode y mantiene un bun flujo de trabajo con la terminal. Pero cuando intentas usar ionic emulate ioslanzará un terrible error.

Error: ios-sim was not found. Please download, build and install version 1.7 or greater from https://github.com/phonegap/ios-sim into your path. Or 'npm install -g ios-sim'; using node.js: http://nodejs.org/

Esto ocurre porque necesitas instalar un paquete llamado ios-sim que permite acceder al emulador de IOS desde la línea de comandos, Se puede instalar con un simple npm install.

$ sudo npm install -g ios-sim

Con ello instalado, serás capaz de emular sobre el simulador de IOS.

Plugins perdidos

Cuando actualizas tu proyecto a una nueva versión de Cordova, a veces obtendrás un error de que no puede localizar los plugin. Abres Xcode, intentas ejecutar y contruir (build) desde allí y obtienes el siguiente mensaje por consola

2014-11-12 14:36:18.284 Tell The DJ[14955:c07] ERROR: Plugin 'Device' not found, or is not a CDVPlugin. Check your plugin mapping in config.xml.
2013-08-23 14:36:18.284 Tell The DJ[14955:c07] -[CDVCommandQueue executePending] [Line 116] FAILED pluginJSON = [
	"Device1096677259",
	"Device",
	"getDeviceInfo",
	[

	]
]

Ésto puede ocurrir por diferentes razones, pero la más común es añadir manualmente los archivos en el proyecto nativo. El proceso de enlazado durante el ciclo de compilación se rompe y ningún archivo nativo se terminan de mover. Para solucionar ésto, borra el archivo ios.jsonde la carpeta de plugin. Esto hará que Cordova tenga que copiar de nuevo todos los archivo de los plugins.

En casos extremos, a veces es necesario eliminar el proyecto.

Directorio de Cordova

Éste es una de los más extremos y frustrantes errores.

Intentas usar algun comando de Cordova/Ionic y obtienes este error. 
Current Working Directory is not Cordova-based project

En este punto no es capaz de hacer nada con su proyecto. ¿Por qué ocure ésto? Por varias razones, pero sobre todo porque se ha corrompido su entorno de desarrollo. Puedes ver un hilo donde se ocurre a diversos usuarios.

Para este caso, la solución fue ejecutar

$ cordova create .

Básicamente una carpeta oculta que debería estar, no estaba. Ejecutanto ese comando se corrige el problema.

Actualizaciones de Cordova

Cuando Cordova lanza una nueva versión, lo mejor es siempre actualizar, así que ejecuta este comando para actualizar la librerías de Cordova.

$ sudo npm update -g cordova

Fantástico, pero cuando intentas hacer algo en un proyecto, obtendrás un error. Ocurre porque hay diferentes versiones de cordova para cada plataforma, así que desde dentro de tu proyecto necesitas ejecutar el comando update de Cordova

$ cordova platform update ios
$ cordova platform update android

Ahora todas tus plataformas están actualizadas.

Windows y Git

Has encontrado un fantástico plugin en Github y quieres instalarlo para usarlo en tu aplicación, vamos a instalarlo.

ionic plugin add http://github.com/user/awesome-plugin

Pero entonces aparece un error. ¿Qué está ocurriendo? Algo que Cordova no menciona es que necesitas tener instalado git para instalar plugins desde Github. Algo así se puede pasar por alto, pero es fácil de solucionar. Solo visita la página de git, instálalo y configúralo y listo.

Hooks de Cordova perdidos

Si tu proyecto fue creado con versiones antiguas del cliente de Ionic, quizás obtengas algo como esto:

addCliHookDirectory fs.chmodSync: Error: ENOENT, no such file or directory 'hooks/after_prepare/010_add_platform_class.js'

Afortunadamente se ha solucionado en las últimas versiones. Si no se encuentra el hook, intenta construir (un build) en cualquiera de las plataformas IOS o Android y el cliente de Ionic lo añadirá de nuevo.
